# 談話室（下部領域）設計

## 概要

縦画面（portrait）時、4:3ゲームエリアの下に表示される装飾エリア。
前作の談話室を踏襲しつつ、最大3体の立ち絵配置とスロットタップ選択UIを導入する。

## 画面比率と表示モード

画面のアスペクト比に応じて3つのモードに分かれる。

| 画面比率 | モード | ゲームエリア配置 | 談話室 |
|---------|--------|---------------|-------|
| 4:3以上（横長） | 横画面モード | 画面中央 | 非表示 |
| 4:3〜2:3の間 | 縦画面（余白小） | **縦方向中央** | 非表示 |
| 2:3以下（縦長） | 縦画面（談話室あり） | 上部配置 | 表示 |

### 判定方法

- **談話室の表示**: `@media (max-aspect-ratio: 2/3)` で判定（CSSメディアクエリまたはTailwindカスタム）
- **ゲームエリアの縦中央配置**: 4:3〜2:3の間では `items-start` → `items-center` に切り替え
- **前作との違い**: 前作は `portrait`（1:1）をしきい値としていたが、今作は2:3に変更。iPadなど正方形に近いデバイスでは談話室を出さない

### 実装イメージ

```
横画面（4:3以上）:
┌────────────────────────────────────┐
│         4:3 ゲームエリア（中央）      │
└────────────────────────────────────┘

縦画面・余白小（4:3〜2:3）:
┌─────────────────────────┐
│          余白             │
│  ┌───────────────────┐  │
│  │ 4:3 ゲームエリア    │  │  ← 縦方向中央
│  └───────────────────┘  │
│          余白             │
└─────────────────────────┘

縦画面・談話室あり（2:3以下）:
┌─────────────────────────┐
│    4:3 ゲームエリア       │  height: 75dvw
│    (GameContainer)       │  背景は4:3内に収める
├─────────────────────────┤
│    談話室エリア            │  height: calc(100dvh - 75dvw)
│    (RoomArea)            │  width: 100dvw
└─────────────────────────┘
```

## 表示条件

- **談話室**: `@media (max-aspect-ratio: 2/3)` のときのみ表示
- 横画面・OBSブラウザソースでは非表示（4:3エリアのみ取り込まれる想定）
- ゲーム画面（4:3内）のアシスタントには影響しない（独立した領域）

### 上画面の背景画像の扱い

- 縦画面時、`body` の背景画像（寮別背景 `bg_*.png` 等）が4:3エリアの外にはみ出さないようにする
- 現状は `background-size: cover` で16:9の画像が画面全体に広がるため、4:3の下にも背景が見えてしまう
- 対策：縦画面時は背景画像を4:3エリア内に収める（`body` ではなく `GameContainer` の疑似要素に設定する等）
- 4:3エリアの外（談話室の背後）には `body` の `background-color` のみが見える形にし、その上に談話室の背景画像を重ねる

## 構成要素

### 1. 背景画像

- ユーザーが**寮を選択して背景を切り替え**できる（タレント選択とは独立）
- 切り替えUI：寮ボタン4つ（エンブレムまたはシンプルなボタン）を談話室エリア内に配置。タイトル画面のときのみ表示
- 背景画像一覧：`bg_wa.png`（バゥ寮）/ `bg_me.png`（ミュゥ寮）/ `bg_co.png`（クゥ寮）/ `bg_wh.png`（ウィニー寮）
- 選択した寮はLocalStorageに保存（`parermaster2_room_talents` 内に含める）
- 初期値：ランダム
- 上画面（4:3ゲームエリア）の寮背景と被る場合がありうるが、一旦許容

### 2. テーブル（手前の遮蔽物）

- 談話室の手前にテーブル状の画像を配置し、タレントがテーブルの裏側に立っているように見せる
- テーブルで立ち絵の下半身を自然に隠すことで、クリップの不自然さを軽減
- **レイヤー順（奥→手前）**: 背景 → 立ち絵 → テーブル → 選択UI・吹き出し
- テーブル画像は要発注（`ui/room_table.png` 等）。横幅100%、高さは談話室エリアの下部30〜40%程度を想定

### 3. 立ち絵スロット（最大3体）

左・中央・右の3スロットに立ち絵を配置できる。

#### 配置

| スロット | 水平位置 | 備考 |
|---------|---------|------|
| 左 | 25% | — |
| 中央 | 50% | 初期状態ではここに1体（前作同様） |
| 右 | 75% | — |

- 立ち絵画像：`kv/orig/{student_id}.png`（833×1500px、アスペクト比 約1:1.8）
- 表示方法：テーブルの裏側に立つ形。立ち絵の下部はテーブルで自然に隠れる
- GSAPアニメーション：前作踏襲（上下移動・左右移動・回転・拡大縮小を各スロット独立で実行）
- 重なり順：左が最背面、中央が中間、右が最前面（z-index制御）

#### 2期生の立ち絵

- 2期生の立ち絵（`kv/orig/`）が存在しない場合、正方形画像（`face/*.png`）で代替表示する
- 代替時はサイズ・配置を調整（正方形なので上半身クリップは不要）

### 4. 吹き出し

- タレントがランダムなタイミングでセリフを表示する（ゲーム画面のアシスタントとは独立）
- 表示中のタレントの中からランダムに1体を選び、その頭上付近に吹き出しを表示
- 一度に表示される吹き出しは**1つまで**

#### 表示タイミング

- 一定間隔（例：15〜30秒ごと）でランダムに出現
- 数秒間表示した後フェードアウト
- ユーザー操作（スロットタップ等）中は出現しない

#### セリフのソース

- **未定（保留）**。プロフィールデータからの自動生成、専用セリフデータ、またはその組み合わせを検討中

#### 演出

- 吹き出しはテーブルより手前（最前面）に表示
- フェードイン・フェードアウトで自然に出現・消失
- 吹き出しの尻尾は発言タレントの方向を向く

### 5. 選択UI（スロットタップ方式）

#### 基本操作

1. **空きスロットをタップ** → タレント選択ドロップダウンが開く
2. **立ち絵をタップ** → 同じドロップダウンが開く（タレント変更 or 解除）
3. **ドロップダウンで「解除」を選択** → スロットが空になる

#### 空きスロットの表示

- 薄い円形 or 枠線で「＋」マークを表示（タップ可能であることを示す）
- 背景に溶け込むよう半透明

#### ドロップダウン

- 前作踏襲の寮別グループ化リスト
- 寮名ヘッダー → タレント一覧（各寮内）
- 先頭に「解除」選択肢を追加
- 他のスロットで選択済みのタレントはグレーアウト（同一タレントの重複配置を防止）
- 1期生・2期生の両方を表示（世代でサブグループ化するかは要検討）

#### タレント選択ボタンの表示タイミング

- **タイトル画面のとき**のみ選択UIを表示（`showSelector` prop、前作踏襲）
- クイズ中・結果画面等では立ち絵のみ表示、選択操作は不可

## データ永続化

- LocalStorage に選択状態を保存
- キー：`parermaster2_room_talents`
- 値：`{ left: string | null, center: string | null, right: string | null, dormitory: string }`（タレントIDまたはnull、寮名）
- 初回アクセス時：中央スロットにランダムな1期生を配置、左右は空、背景の寮はランダム

## 実装方針

### コンポーネント構成

```
RoomArea（メインコンテナ）
├── 背景画像
├── TalentSlot × 3（左・中央・右）
│   ├── 立ち絵画像 + GSAPアニメーション
│   └── 空きスロット表示（＋マーク）
├── テーブル画像（立ち絵の手前）
├── SpeechBubble（吹き出し、テーブルの手前）
└── TalentSelector（ドロップダウン、1つだけ表示）
```

### ファイル配置

- `src/features/room/` ディレクトリに配置
  - `RoomArea.tsx` — メインコンテナ
  - `TalentSlot.tsx` — 立ち絵スロット（アニメーション含む）
  - `SpeechBubble.tsx` — 吹き出し表示
  - `TalentSelector.tsx` — ドロップダウン選択UI
  - `useRoomStore.ts` — Zustand ストア（スロット選択状態 + LocalStorage永続化）

### 配置場所

`src/app/App.tsx` で GameContainer の直後に配置。CSS `.room-visible` で 2:3 以下のときのみ表示：

```tsx
<div className="room-visible">
  <RoomArea showSelector={screen === 'title'} />
</div>
```

## 前作との差分

| 項目 | 前作 | 今作 |
|------|------|------|
| 立ち絵数 | 1体 | 最大3体 |
| 選択方法 | 右上ドロップダウン1つ | スロットタップ → ドロップダウン |
| 「談話室のみ」モード | あり（寮ごと） | なし（スロットを全て解除すれば背景のみ） |
| 背景の決定 | 選択タレントの寮 | 中央→左→右の優先順で寮決定 |
| 2期生対応 | なし（1期生のみ） | あり（立ち絵未配置時は正方形画像で代替） |
| テーブル | なし | 手前にテーブルを配置、立ち絵の下半身を自然に隠す |
| 吹き出し | なし | タレントがランダムにセリフを表示（セリフ内容は保留） |
| データ保存 | タレントID 1つ | 3スロット分のオブジェクト |
