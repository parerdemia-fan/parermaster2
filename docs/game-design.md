## 1. 設定画面の構成

### 画面遷移

出題範囲はパターン2（世代別と寮別を独立）を採用（→ [specification.md](specification.md) §3 参照）。

```
タイトル画面 → 「1期生編」or「2期生編」or「寮別モード」→ 設定画面 → ゲーム開始
```

│  └──────────┘└──────────┘ │

**知識クイズ（世代別のみ。寮別モードでは提供しない）**
```
┌──────────────────────────┐

│                            │
│ ── ゲーム ──               │
│ ○おぼえよう ○顔名前当て   │
│ ●知識クイズ                │
│                            │
│ ── 難易度 ──               │
│ ●ふつう ○むずかしい 🔒激ムズ     │
│                            │

└──────────────────────────┘
```
- 問題数は30問固定（問題プールが30問未満の場合は全問出題）。問題数選択UIは廃止
- 2期生の知識クイズは3段階難易度（きほん/ふつう/むずかしい）。むずかしいは準備中表示。詳細は [question-types.md](question-types.md) 参照
- 激ムズは該当スロットのシルバーバッジ獲得で解放（顔名前当てと同じ）

### プレイヤー名
- デフォルト: 「リスナー」
- 設定画面の下部に「あなたの名前: リスナー [変更]」の形式で配置
- 初回起動時に入力を促す演出はしない（すぐ遊べることを優先）
- 使用箇所: 開発日誌（手記エントリ）、応援演出、結果画面、Xシェア文言、タイムアタック記録

### 設定の保持
- 前回の設定をLocalStorageに保存し、次回起動時に復元
- リピーターはスタートボタンだけで即開始可能

### クイズ画面ヘッダー

ゲームプレイ中の画面上部にヘッダー領域を設け、問題の情報を表示する。全問題タイプで共通のコンポーネントとして実装する。

```
┌──────────────────────────────────────────────────────────┐
│ 📸 顔当て                                                │
│ ★☆☆☆☆   この子はどれ？        (5/20) [アシスタント]    │
└──────────────────────────────────────────────────────────┘
```

**表示項目:**
- 左上: 問題タイプラベル（📸 顔当て / ✏️ 名前当て / 🧩 名前を作ろう / 💡 知識クイズ）
- 左上ラベルの下: 問題難易度（★表示、0.5刻み、最大5）
- 中央: 問題文テキスト（この子はどれ？ / クイズに挑戦！ 等）
- 右: リング状プログレス（SVG円環、中央に分数表示）+ アシスタント

**アシスタント:**
- 1問ごとにランダムな1期生が表示される
- 選択肢に含まれるタレントはアシスタントから除外
- セリフバブル（正解/不正解に応じたコメント）+ 名前ラベル + 正方形顔画像

**レイアウト方針:**
- 高さ14cqmin、半透明黒背景にぼかし
- 問題タイプラベル+★は左上に縦並び、問題文は中央に大きく表示
- アシスタントはヘッダー右端に下揃えで配置

### 知識クイズ画面レイアウト

他の問題タイプと同じ左右分割レイアウトを採用する。

```
┌──────────────────────────────────────────────────────────┐
│ [ヘッダー]                                                │
├────────────────────────┬─────────────────────────────────┤
│                        │                                 │
│   問題文パネル          │   選択肢（4行 or 2×2グリッド） │
│  （ダーク半透明）       │   （カラフルパターン付き）      │
│                        │                                 │
│  ── 回答後 ──          │                                 │
│   解説テキスト          │                                 │
│   解説画像              │                                 │
│   情報源リンク          │                                 │
│                        │                                 │
├────────────────────────┴─────────────────────────────────┤
│ [← 戻る]                        [← 前へ] [次へ →]       │
└──────────────────────────────────────────────────────────┘
```

**左側（45%）: 問題文 + 解説パネル**
- ダーク半透明背景（`rgba(0,0,0,0.45)` + blur）に白文字
- 回答前: コンテンツに合わせた高さで上寄り中央に配置
- 回答後: 解説がアニメーションで展開。パネルが大きくなり、はみ出した分はパネル内スクロール
- 解説がない問題では回答後もパネルサイズは変わらない

**右側（48%）: 選択肢**
- テキスト選択肢: 名前当てと同じカラフルなパターン付きボタン（4行、高さ13cqmin固定）
- タレント選択肢: 顔当てと同じ2×2グリッド（顔画像 + 名前ラベル）

## 3. アチーブメント

### 概要
- 前作（27個、個別画像必要）よりシンプルな構成
- 画像は少数（10枚以内）で、進化段階はCSSエフェクト（光沢・キラキラ・サイズ変化）で表現
- 前々作のように同じバッジが進化していく方式

### バッジ・称号の画像

画像一覧は [unlock-conditions.md](unlock-conditions.md) を参照。画像アセットの詳細は [image-assets.md](image-assets.md) を参照。

### バッジ獲得条件

- **全問正解**が必須条件
- **顔名前当て**: 全問題タイプ（顔当て/名前当て/名前を作ろう）が出題される
- **知識クイズ**: 問題数30問（不足時は全問）で全問正解

### バッジの進化段階

全問正解で獲得。難易度に応じて同じ画像が進化（上位で上書き）：

| レベル | 条件 | 見た目の変化 |
|--------|------|-------------|
| ブロンズ | ふつうで全問正解 | 小さい・控えめ |
| シルバー | むずかしいで全問正解 | 中サイズ・光沢エフェクト |
| ゴールド | 激ムズで全問正解 | 大きい・キラキラエフェクト |

### 激ムズの解放条件

- 該当スロットで**シルバーバッジを獲得**（= むずかしいで全問正解）すると激ムズが解放される
- スロットごとに独立して解放される（例: 2期生・バゥ寮のシルバー獲得で、2期生・バゥ寮の激ムズのみ解放）
- 知識クイズも同様（1期生知識クイズのシルバー獲得で激ムズ解放）

### バッジスロット一覧（計8スロット）

**2期生エリア（2スロット）:**

| スロット | 種類 | 進化段階 |
|---------|------|---------|
| 2期生・全員 | クリアバッジ | ブロンズ→シルバー→ゴールド |
| 2期生・知識クイズ | 知識バッジ | ブロンズのみ（難易度なしのため）。高難易度追加時にシルバー・ゴールドへ進化可能に |

**1期生エリア（2スロット）:**

| スロット | 種類 | 進化段階 |
|---------|------|---------|
| 1期生・全員 | クリアバッジ | ブロンズ→シルバー→ゴールド |
| 1期生・知識クイズ | 知識バッジ | ブロンズ→シルバー→ゴールド |

**寮別エリア（4スロット）:**

| スロット | 種類 | 進化段階 |
|---------|------|---------|
| バゥ寮（1期生+2期生） | クリアバッジ | ブロンズ→シルバー→ゴールド |
| ミュゥ寮（1期生+2期生） | クリアバッジ | ブロンズ→シルバー→ゴールド |
| クゥ寮（1期生+2期生） | クリアバッジ | ブロンズ→シルバー→ゴールド |
| ウィニー寮（1期生+2期生） | クリアバッジ | ブロンズ→シルバー→ゴールド |

### 総合称号（複合アチーブメント）

| 称号 | 条件 | 演出 |
|------|------|------|
| 2期生マスター | 2期生エリア全スロットが最大レベル（知識クイズはブロンズ、全員はゴールド） | 特別演出 |
| 1期生マスター | 1期生エリア全スロットがゴールド | 特別演出 |
| パレ学マスター | 両方達成 | スタッフロール解放。シークレット称号（達成まで非表示） |
| パレ学グランドマスター | パレ学マスター + タイムアタック7分21秒以内クリア | パレ学マスターのアップグレード。シークレット称号（達成まで非表示） |

### 解放トリガー

| 条件 | 解放されるもの |
|------|-------------|
| 世代別4スロット（2期生・1期生の全員+知識クイズ）が最大レベル | タイムアタック解放 |

※ 寮別エリアの4スロットはタイムアタック解放条件に含まない
※ 「最大レベル」= 2期生知識クイズはブロンズ、他はゴールド。将来的に2期生知識クイズに高難易度が追加された場合、その難易度クリアで獲得できるバッジを条件とする

### その他のゲーミフィケーション要素

| 要素 | 方針 |
|------|------|
| スコア・正解率 | 結果画面に表示（前作踏襲） |
| Xシェア | 結果画面から投稿（前作踏襲） |
| スタッフロール | パレ学マスター称号で解放（§4 スタッフロール参照） |
| 開発日誌 | AIが丁寧語で執筆。開発記録＋プレイヤーの行動が反映される手記エントリを統合（§4参照） |

## 4. ストーリー・演出要素

### 基本方針
- ゲーム全体を貫くストーリーや、タレント個人に台詞を喋らせるような演出は行わない
  - タレントから苦情が来るリスクを避ける
- その分、クリア演出を豪華にする方向で差別化する

### クリア演出（全問正解時）
- 特別な演出（紙吹雪、バッジ進化アニメーション等）
- 応援タレントが祝福してくれる演出
- スキップ可能

### 応援演出
- 2期生の問題出題中に1期生がランダムに画面端で応援
- 1期生の問題出題中に2期生がランダムに画面端で応援
- 台詞は使わず、立ち絵のみ（またはシンプルなリアクション）

### 開発日誌

開発記録と、プレイヤーの行動が反映される手記エントリを1つの画面に統合する。

**開発記録エントリ（静的）:**
- AI（Claude Opus 4.6）が丁寧語で執筆する実際の開発記録
- データソース: diary.json
- 最初から全エントリ閲覧可能（隠し要素にしない）
- リアルタイムの日付で、実際の開発過程を記録
- 日付は暦日ではなく「寝るまで」を1日とする（深夜0時を過ぎても作業継続中なら前日の日付）

**手記エントリ（動的・パーソナライズ）:**
- ゲーム進行のマイルストーンごとに1エントリずつ解放される
- テンプレート＋実データの差し込みにより、プレイヤーごとに内容が異なる
- データソース: LocalStorage（マイルストーン達成時に生成・保存）
- 一度生成されたエントリは固定（再プレイで上書きしない）
- 開発記録と見た目で区別する（アイコンや背景色等）

**手記エントリ例（イメージ）:**
```
（初回プレイ完了時）
「今日、初めてのプレイヤーが来てくれました。最初に挑戦したのは
 ◯◯寮の顔名前当て。初めて正解した2期生は◯◯◯◯◯でした。
 この名前は、きっとずっと覚えていてくれるのではないでしょうか。」

（タイムアタック初挑戦時）
「ついにタイムアタックに挑んでもらえました。記録は◯分◯秒。
 106人全員と向き合った◯◯分間は……」

（パレ学マスター達成時）
「106人全員の顔と名前を覚えた人がここにいます。……」
```

**表示方式:**
- 開発記録エントリと手記エントリを日付順にマージして表示
- 手記エントリは解放された時点の日付で挿入される
- エントリ数・トリガー条件はアチーブメント設計確定後に決定
- ストーリーの詳細な内容は開発後半で執筆

### スタッフロール

パレ学マスター称号で解放される。前作のスタッフロールを踏襲しつつ、2nd Season の体制に合わせて更新する。

**表示タイミング:**
- **初回自動表示**: パレ学マスター称号獲得直後にタイトル画面を開いたとき、自動でスタッフロールが再生される。表示済みフラグを LocalStorage に保存し、2回目以降は自動再生しない
- **手動表示**: パレ学マスター称号を持っている場合、「このゲームについて」画面にスタッフロール再生ボタンを表示する

**画面仕様:**
- 黒背景（`rgba(0,0,0,0.95)`）のオーバーレイとして GameContainer 上に表示
- タイトル → 役職＋メンバーのリスト → Special Thanks → Thank you for playing! の順に下から上へスクロール
- 右上に Skip ボタンを常時表示し、いつでも閉じられる

**Special Thanks:**
- プレイヤー名が「リスナー」（デフォルト）の場合: 「プレイしてくれたあなた」と表示
- プレイヤー名がデフォルト以外の場合: そのプレイヤー名を表示

**データ:**
- `public/data/staff.json` に `{ role: string, members: string[] }[]` 形式で定義
- Special Thanks のプレイヤー名部分は表示時に動的に差し込む（JSONにはプレースホルダを含めない）

### 結果画面の構成

```
┌──────────────────────────┐
│         結果発表           │
│                            │
│   正解数: 14 / 15          │
│   正解率: 93.3%            │

```

- タイムアタックの場合はクリアタイムも表示
- アチーブメント（バッジ）を獲得した場合はそれ用の表示を行う

### UI/UXの方針

#### UIパーツ
- ボタン: 3-patch画像（左端・中央リピート・右端）
- パネル: 単一画像（CSS伸縮）
- ボタンラベルは原則テキスト描画（必要に応じて画像ラベルも可）

#### レイアウトのバランス指針（設定画面の実装で確立）

- **見出し・ラベルは背景に埋もれさせない。** 半透明パネル上のテキストであっても、薄いグレー（`#999` 等）は背景と同化して目立たない。ボタンより控えめにしつつも、はっきり読める濃さ（`#555` 程度）を確保する
- **セクション間の余白はボタン内の gap より明確に広くとる。** セクションの区切りが視覚的に伝わるよう、見出し上の余白をボタン間 gap の1.5倍以上にする。ただしパネル先頭のセクションは上余白を詰めて、パネル内側に無駄な空白を作らない
- **主要アクション（スタートボタン等）の前には他のセクション間より広い余白を置く。** 選択肢群とアクションの間に視覚的な「溜め」を作ることで、ボタンの存在感と押しやすさを高める
- **半透明パネルは透過度を十分に確保する。** 不透明度が高すぎる（0.85等）と背景が透けず白い板に見えてしまう。背景画像が透けて見える程度（0.55程度）に抑え、blur で可読性を補う
- **cqmin で配置する要素は縦方向の合計を検算する。** 4:3横長画面のコンテナ高さは約75cqmin。各要素の高さ + gap + margin を積み上げて75cqmin以内に収まることを確認してからサイズを決める。「見た目がよさそうな値」を個別に決めるとはみ出す

#### フォント
- M PLUS Rounded 1c（前作踏襲）
- CSS Container Query単位（cqmin）によるサイズ制御（前作踏襲）

#### 背景画像
- 出題画面の背景は出題範囲に応じて切り替える
  - 寮別: 寮ごとの背景（1期生・2期生共通）
  - 全員（1期生）: 1期生用の背景
  - 全員（2期生）: 2期生用の背景
  - タイムアタック/ハードコア: 専用背景 or 問題ごとに切り替え（未確定）
- 実装方式: `body::before` 疑似要素で背景レイヤーを分離し、CSS変数（`--bg-image`, `--bg-blur`, `--bg-blur-scale`）で動的制御
- クイズ/結果画面では背景に軽いぼかし（3px）を適用し、手前のキャラクター・UIを引き立たせる
- 寮別背景が未作成の場合はデフォルト背景（bg_title）にフォールバック

#### 選択肢レイアウト（基本形、前作踏襲）
- 画像選択肢: 正方形で2×2グリッド
- テキスト選択肢: 横長ボタン
- 問題タイプによっては上記以外のレイアウトになる（名前を作ろう等）

#### デザインの方向性

全体のテイスト: **明るく、カラフルで、ポップなカジュアルゲーム**。前作のダークファンタジー風とは対照的に、温かみのある明るいトーンで統一する。

**カラーパレット:**
- 背景: 暖色系（ベージュ、薄ピンク、淡いオレンジ）。柔らかく、ぼかしのかかった室内・屋外の風景
- インタラクティブ要素: 彩度の高いビビッドカラー（ピンク、オレンジ、グリーン、ブルー、レッド）
- パネル・情報表示: 白〜半透明白。背景との高コントラストを確保
- テキスト: 濃色（ダークグレー〜黒）が基本。ボタン上は白

**形状・質感:**
- 角丸を全面的に使用（ボタン、パネル、タイル、バッジすべて）
- 柔らかい影（subtle drop-shadow）による控えめな立体感
- グラデーションは穏やかに（飴玉・マカロンのような光沢感）
- 硬い直線・シャープな角は使わない

**ボタン:**
- ゲーム開始ボタン（タイトル画面）: 3-patch、丸薬型、色で1期生/2期生を区別
- アクションボタン: 角丸の大きめボタン、ビビッドカラー背景 + 白テキスト
- 選択肢タイル: カラフルな角丸正方形。色はランダムまたはカテゴリ別

**パネル・カード:**
- 白〜半透明白の角丸パネル
- 問題文・情報表示は白パネル上に配置して可読性を確保
- ラベルバッジ（「名前を作れ！」等）は色付き角丸で左上などに配置

**キャラクター表示:**
- 立ち絵はゲーム画面内に大きく配置（左側など）
- アシスタントキャラは右上に小さく配置 + 吹き出し

**画面全体の印象:**
- カジュアルゲーム・知育アプリに近いポップさ
- 情報の整理がされていて見やすい
- 色の使い方でインタラクティブ要素と情報表示を明確に区別

**デザイン素材の生成:**
- Gemini（Nano Banana 2 / Gemini CLI）を活用
