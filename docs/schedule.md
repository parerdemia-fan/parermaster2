## 1. リリース方針

- **2期生お披露目**: 2026年4月中旬と予想（前年の1期生入寮式が4月18日）
- **リリース目標**: お披露目当日〜3日以内
- **戦略**: お披露目前に2期生データ以外のすべてを完成させ、お披露目後はデータ流し込みのみで即リリース

## 4. Phase 4: 2期生データ投入スプリント（お披露目後）

お披露目で判明する情報を最速で収集・投入する。

| 作業 | 所要時間（目安） | 依存 |
|------|---------------|------|
| 2期生60名の名前・寮・画像の収集 | 数時間 | 公式発表 |
| talents.json への投入 | 1〜2時間 | 上記 |
| タレント画像の加工・配置 | 2〜4時間 | 上記 |
| 2期生知識クイズ問題の作成（共通問題は流用済） | 並行作業 | 寮構成確定 |
| 2期生全体の動作テスト | 1〜2時間 | 上記すべて |
| デプロイ | 数分 | 上記 |

**当日リリースのための前提条件:**
- Phase 3終了時点で1期生側が完全に動作している
- 2期生データの投入手順がドキュメント化・テスト済みである
- 画像加工のパイプライン（リサイズ・WebP変換等）が自動化されている

## 5. MVP範囲（初期リリース）とポストリリース

### MVP（初期リリースに含めるもの）

| 機能 | 理由 |
|------|------|
| 問題タイプ1・2・4（顔当て・名前当て・知識クイズ） | 最重要。これだけで「顔と名前を覚える」+「深く知る」目的を達成できる。知識クイズは1期生分を前作から流用、2期生は共通問題で初期提供可能 |
| アチーブメント（12スロット＋進化） | ゲーミフィケーションの柱 |
| タレント一覧・詳細 | 学習の補助 |
| 設定画面・結果画面 | 基本機能 |
| Xシェア | 拡散の柱 |
| 応援演出 | 1期生↔2期生の交流感。立ち絵のみなので低コスト |
| 案内 | 初回プレイヤー向け |

### ポストリリース（段階的に追加）

| 機能 | 優先度 | 理由 |
|------|--------|------|
| 問題タイプ3・5〜8（名前を作ろう、プロフィール→顔当て、名前はどこ？等） | 高 | リリース後1〜2週間で順次追加。飽き防止 |
| タイムアタック | 中 | 全スロットゴールドが条件なので、初日に到達するプレイヤーはいない |
| 開発日誌 | 中 | 隠し要素 |
| ショートストーリー | 低 | 問題作成・レビューに時間がかかる |
| スタッフロール | 低 | パレ学マスター称号が条件。最後に追加で間に合う |
| 追加問題タイプ（タイムアタック専用含む） | 低 | タイムアタック解放後に価値が出る |

## 6. リスクと対策

| # | リスク・課題 | 影響度 | 対策 |
|---|-------------|--------|------|
| 1 | 2期生お披露目が予想より早い | 高 | 問題タイプ1・2・4だけでもリリース可能な設計にする |
| 2 | 2期生の寮構成・人数が想定と異なる | 高 | 寮数・人数をハードコードしない |
| 3 | お披露目当日に全情報が揃わない | 中 | 判明分で先行リリース→追加アップデート |
| 4 | 画像素材の確保・権利関係 | 高 | 二次創作ガイドライン遵守 |
| 5 | 問題数の確保（特に2期生知識クイズ） | 中 | 共通問題を前作から流用して初期対応 |
| 6 | SEOの失敗（前作・前々作で未達） | 中 | OGP・メタタグ・構造化データを初期リリースから対応 |
| 7 | お披露目が遅い（5月以降） | 低 | 追加問題タイプの開発に時間を使える。むしろ好都合 |
| 8 | 2期生が60名でない | 中 | 人数もハードコードしない |
