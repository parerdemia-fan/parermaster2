# 相性診断

プレイヤーの性格傾向から相性の良い寮生を診断する。解放条件なし（最初から利用可能）。

## 導線

タイトル画面の右下サブメニューアイコンから遷移。アイコンの並び順（上から）：

1. 寮生一覧
2. 相性診断
3. 実績
4. 案内

---

## 概要

- 10問の性格診断（各問4択）に回答すると、相性の良い寮生が表示される
- タレントに台詞を喋らせない（地の文・UIのみで構成）
- Xシェア対応

---

## 性格軸（5軸、各0〜10）

| 軸 | 低い(0) | 高い(10) |
|---|---|---|
| テンション | おっとり・癒し系 | ハイテンション・にぎやか |
| 興味 | カルチャー系（アニメ・ゲーム・音楽鑑賞） | リアル系（スポーツ・料理・旅行） |
| 思考 | 感性・直感で動く | 論理・分析で動く |
| スタンス | マイペース・独立 | 協調・面倒見がいい |
| 表現 | 受け手（鑑賞・消費） | 作り手（創作・発信） |

各軸は独立性が高い：
- テンション × スタンス: ハイテンだけど一匹狼、おっとりだけど面倒見がいい
- 興味 × 思考: オタクだけど論理派（考察勢）、リアル趣味だけど感覚派
- 表現 × テンション: 控えめだけど作品は大胆、にぎやかだけど創作はしない

---

## タレントプロファイリング

各タレント（70人）に5軸のスコア（0〜10）を設定する。スプレッドシートで管理し `pnpm sync` で取り込む。

### 情報源

| ソース | 内容 | 主な対象 |
|---|---|---|
| `talents.json` | 趣味・特技・好きなもの・夢・MBTI・自己紹介・tone（話し方） | 全70人 |
| `questions.json` | テキストクイズ（タレントの深掘り情報） | 主に1期生 |
| `awards.json` | イベント入賞歴174件・28種類 | 主に1期生 |

### 各軸の判定材料

| 軸 | talents.json | questions.json | awards.json |
|---|---|---|---|
| テンション | intro、tone、firstPerson、MBTI(E/I) | 行動傾向に関する問題 | — |
| 興味 | hobbies、favorites | 趣味に関する問題 | イベント種類（グルメ、旅行等） |
| 思考 | skills、MBTI(T/F, S/N) | 知識・考察系の問題 | 学力テスト順位、雀魂大会 |
| スタンス | hobbies、dream、MBTI(J/P) | 対人関係の問題 | 投票イベント成績（ファンとの関係性） |
| 表現 | skills、hobbies | 創作活動の問題 | オリジナル曲制作、アイキャッチ、歌ってみた |

### 注意事項

- 2期生はデータ蓄積が少ないため、talents.json のプロフィールが主な判定材料になる
- MBTIは70人中36人のみ（2期生は全員空）。あくまで補助情報として使う

## 質問設計（10問4択）

### 方針

- 学園生活をテーマにした親しみやすい質問
- 各選択肢が2軸にスコアを加算
- 10問 × 2軸 = 各軸に平均4回のデータポイント
- 「どの軸を測っているか」が回答者にバレにくい設計にする

### 質問例

> Q. 学園の文化祭。あなたがやりたいのは？
> - ステージでパフォーマンス → テンション+3, 表現+2
> - 展示の企画・装飾 → 表現+3, 思考+1
> - 実況・MC担当 → テンション+2, スタンス+2
> - 裏方で運営を支える → 思考+3, スタンス+2

> Q. 配信を見るならどんな内容？
> - ゲーム実況 → 興味(カルチャー寄り)+3, テンション+1
> - 雑談・相談枠 → スタンス+3, テンション+1
> - 歌枠・ライブ → 表現+3, テンション+2
> - 作業・勉強配信 → 思考+2, 興味(カルチャー寄り)+1

※ 質問・選択肢・スコア配分の詳細は実装時に設計する

---

## マッチングアルゴリズム

1. 5軸のスコアを全て0で初期化
2. 各質問の選択肢に設定された加算値（+1〜+3）をスコアに加算（マイナスなし）
3. 10問回答後のスコアベクトルと、各タレントの5軸ベクトルのコサイン類似度を計算
4. 類似度上位3人を「相性の良い寮生」として表示

※ 加算のみの設計により、多く加算された軸が高く、加算されなかった軸は相対的に低くなる。コサイン類似度はベクトルの「形」を比較するため、絶対値の大小ではなく軸間の比率でマッチングされる。

## データ設計

タレントの性格スコアは `public/data/personality.json` に独立ファイルとして管理する（talents.json とは分離）。

```json
{
  "25CO002": {
    "tension": 5,
    "interest": 2,
    "thinking": 4,
    "stance": 5,
    "expression": 4
  }
}
```

質問データは別途 JSON（または定数ファイル）で管理する。

```json
{
  "question": "学園の文化祭。あなたがやりたいのは？",
  "choices": [
    { "text": "ステージでパフォーマンス", "scores": { "tension": 3, "expression": 2 } },
    { "text": "展示の企画・装飾",       "scores": { "expression": 3, "thinking": 1 } },
    { "text": "実況・MC担当",           "scores": { "tension": 2, "stance": 2 } },
    { "text": "裏方で運営を支える",     "scores": { "thinking": 3, "stance": 2 } }
  ]
}
```
