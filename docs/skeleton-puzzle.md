# スケルトンパズル

タレント名のひらがなをグリッドに配置するスケルトンパズル（クロスワード形式）。

## パズルの種類

| 種類 | ワード数 | グリッド | 備考 |
|------|---------|---------|------|
| 1期生版 | 46名 | 19×30 | |
| 2期生版 | 23名 | 13×20 | 最もコンパクト |
| 全員版 | 69名 | 23×37 | 最大のグリッド |

## ルール

- 文字はすべてひらがな、点(・)なし
- ヒント文なし。すべて自力で配置する
- 文字入力ではなく、タレントの顔一覧から選んで配置する
- 間違った配置でも即エラーにはしない。ただし交差点の文字が矛盾する場合は視覚的に警告する

> 初期開示ワード（最初から埋まっているマス）の機構自体は実装に存在する（`PuzzleData.initialRevealedWordIds` / 問題テキストにひらがな文字を書くと自動検出される）が、現在配布中のパズルデータでは使用していない。

## 隠しメッセージ

- グリッド内の特定セルに番号(1,2,3...)が振られている
- 番号順に文字を繋げるとメッセージになる
- メッセージ内容は各パズルで異なる

## クリア判定（段階的）

1. **一次クリア**: 番号付き文字の隠しメッセージが正しく完成 → 演出（メッセージ表示）
2. **完全クリア**: 全ワードが正しく配置 → 追加演出

## 画面構成

### アクセス経路

案内画面（AboutScreen）からリンク。解放条件なし。

### 画面レイアウト

- ヘッダー: 戻るボタン、タイトル「スケルトンパズル」、リセットボタン
- メインエリア: パズルグリッド（画面の大部分を占有）
- フッター: 隠しメッセージバー（番号付き文字の進捗表示）

### パズル選択

スケルトン画面に入ると種類選択画面（1期生／2期生／全員のボタン）が表示される。選択するとそのパズルの進捗がロードされ、プレイ画面に遷移する。

### グリッド操作

1. 空きワードスロット（連続する白マス）をタップ
2. 交差点では方向選択（よこ/たて）のポップアップを表示
3. タレント選択パネル（TalentPicker）がオーバーレイ表示

### タレント選択パネル（TalentPicker）

- 寮タブで切り替え（バゥ寮/ミュゥ寮/クゥ寮/ウィニー寮）
- 顔画像付きの一覧表示
- 選択されたスロットの文字数に一致しないタレントはグレーアウト（表示はする）
- 配置済みタレントには配置済みマークを表示
- 配置済みタレントを再選択すると入れ替え

### グリッドのズーム

大きなグリッド（1期生版・全員版）ではピンチズーム・パン操作に対応する。

## 進捗の保存

- localStorage に配置済みワード・クリア状態を保存
- パズル種類ごとに独立（`parermaster2_skeleton_gen1` 等）
- リセットボタンで進捗をクリア（確認ダイアログあり）

## データ形式

パズルデータは `public/data/puzzles/{variant}.json` に配置。

```typescript
interface PuzzleData {
  variant: 'gen1' | 'gen2' | 'all'
  rows: number
  cols: number
  grid: (string | null)[][]             // null=黒マス、string=ひらがな1文字
  words: PuzzleWord[]
  numberedCells: { order: number; row: number; col: number }[]
  hiddenMessage: string
  initialRevealedWordIds: number[]
}

interface PuzzleWord {
  wordId: number
  talentId: string | null               // null = タレントに紐付かない特殊ワード（拡張用、現状未使用）
  hiragana: string
  startRow: number
  startCol: number
  direction: 'across' | 'down'
  length: number
}
```
