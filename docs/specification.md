## 1. プロジェクト概要

### 1.1 プロジェクト名
パレ学マスター 2nd Season（仮）

### 1.2 コンセプト

パレデミア学園のタレントを「知る・覚える・好きになる」ためのWebクイズゲームの続編。

> **用語の統一**
> - ゲームを遊ぶ人 → **プレイヤー**
> - パレデミア学園所属VTuber → 内部（コード・ドキュメント）では **タレント**
>   - ゲーム画面上（UIテキスト）では「タレント」「生徒」は使わない
>   - **この子** ＞ **寮生** ＞ **パレ学生** の優先順で、文脈に合うものを選ぶ

#### 目的（優先順位順）

1. **2期生の顔と名前を楽しく覚えてもらう**（最重要）
   - 2nd Seasonの主役は2期生
   - 新たなパレ学ファンにも、既存ファンにも2期生を知ってもらう入口となる
2. **新たなパレ学ファンに1期生の顔と名前も覚えてもらう**
   - 2期生から入った新規ファンにも1期生を知ってもらう
3. **1期生のことをもっと深く知ってもらう**
   - 前作の深堀り問題（難易度1〜5の4択クイズ）を知識クイズとして継続・拡充
4. **2期生のことをもっと深く知ってもらう**
   - パレ学・寮に関する共通問題は前作から流用し、初期リリースから提供
   - 2期生固有の深堀り問題は十分な問題数を用意できるまで保留

#### 前作との位置づけ
- 前作「パレ学マスター」は現状のまま維持し、問題追加などは行わない
- 2nd Seasonは別アプリとしてリリースする
- 2nd Seasonの1期生対象は**46名**

### 1.3 ターゲットプレイヤー
- パレデミア学園のファン
- 前作・前々作のプレイ経験者
- 2期生から入った新規ファン
- タレント自身（配信でのプレイ、1期生が2期生を覚える、2期生が1期生や同期を覚える）

### 1.4 シリーズの経緯

#### 前々作「パレクイズ」

- ソース: ../parerquiz/
- 技術: React 19 + TypeScript + Vite 6 + Tailwind CSS v4、npm
- 特徴:
  - 名前当て・顔当てモードの2種
  - 4段階の難易度（ふつう / むずかしい / 寮生専用 / 鬼）
  - 60名の1期生が対象
  - AIストーリー要素（61人目のAI、問題進行に伴うメッセージ、名前命名イベント）
  - バッジシステム（LocalStorage永続化）
  - バッドエンドシステム
  - 古いAI残留プロセス（隠し要素）

| 1 | 顔名前当て | 各問題タイプ（顔当て・名前当て等）が規定数ずつ出題される | 継続 / 大幅拡張 | 最高（MVP） |

## 3. 対象タレントと出題範囲

### 対象タレント
- 1期生: 46名
- 2期生: 人数未定（お披露目後に確定）
- 寮構成: バゥ寮 / ミュゥ寮 / クゥ寮 / ウィニー寮（1期生・2期生とも同じ4寮）
- 寮の表示順: バゥ → ミュゥ → クゥ → ウィニー（全画面で統一）

### 出題範囲

2期生の人数が想定より少ない場合、寮ごとの人数が不足し顔名前当ての難易度バランスが崩れる可能性がある。これに対応するため、以下の2パターンのどちらにも対応できる設計とする。お披露目後に確定する。

**パターン1: 世代内で寮別を選択（現行プラン）**

| # | 出題範囲 | 備考 |
|---|----------|------|
| 1 | 2期生・寮ごと（4寮） | 2期生を寮単位で覚える入口 |
| 2 | 2期生全員 | |
| 3 | 1期生・寮ごと（4寮） | |
| 4 | 1期生全員 | |

- タイトル画面: 「1期生編」「2期生編」→ 設定画面で寮を選択
- 1期生と2期生の混合モードは設けない

**パターン2: 世代別と寮別を独立（2期生の人数が想定より少ない場合の代替案）**

| # | 出題範囲 | 備考 |
|---|----------|------|
| 1 | 1期生全員 | |
| 2 | 2期生全員 | |
| 3 | バゥ寮全員（1期生+2期生） | 寮の先輩後輩をまとめて覚える |
| 4 | ミュゥ寮全員（1期生+2期生） | 同上 |
| 5 | クゥ寮全員（1期生+2期生） | 同上 |
| 6 | ウィニー寮全員（1期生+2期生） | 同上 |

- タイトル画面: 「1期生編」「2期生編」+「寮別モード」→ 寮別は設定画面で寮を選択
- 寮別モードは1期生+2期生混合のため、寮あたりの人数が十分に確保される

**共通の設計方針:**
- 出題範囲のフィルタは `generation`（世代）と `dorm`（寮）の組み合わせで表現し、どちらも省略可能とする
- クイズロジックは「対象タレントの配列」を受け取るだけで、フィルタの構成に依存しない
- アチーブメントのスロット定義はデータ駆動とし、パターン確定後に定義を差し替えるだけで対応可能にする

## 4. 難易度・問題数

### 顔名前当て
- 難易度3段階（ふつう / むずかしい / 激ムズ）
  - 激ムズは解放条件あり（[game-design.md](game-design.md) アチーブメント参照）
- 解放済みの各問題タイプが規定数ずつ出題される
- 出題範囲の全タレントが出題対象
- 各問題タイプの難易度別仕様は [question-types.md](question-types.md) を参照

### 知識クイズ
- 問題数: 30問固定（問題プールが30問未満の場合は全問出題）
  - 問題数選択UIは廃止。リピーターが即開始できるシンプルな構成を優先
- 難易度は1期生/2期生で異なる:
  - **1期生**: 3段階（ふつう=difficulty 0 / むずかしい=difficulty 1-2 / 激ムズ=difficulty 3-5）
  - **2期生**: 難易度選択なし（問題数が少ないため1段階）
    - 2期生固有の深堀り問題が増えた段階で難易度を追加

## 5. 非機能要件

### 対応プラットフォーム
- Web（GitHub Pages）
- モバイルブラウザ対応
- PWAは不採用（前作で採用したが、マニフェスト記述の複雑さ・iPhoneでの横画面強制不可・画面見切れ問題・機種差など、旨味に対してコストが高すぎたため）
- 全画面表示はFullscreen APIで対応（Android・PC向け）

### 画面設計方針

#### 基本方針
- **横画面推奨**。タイトル画面に横画面推奨の表示を行う（前作踏襲）
- ゲームエリアは**4:3アスペクト比固定**（前作踏襲）
  - 前作の実績あり、配信で見やすい
  - CSS Container Query単位（cqmin）による一貫したサイズ制御（前作踏襲）
  - 各エリアのサイズは内容に関わらず固定（前作踏襲）

#### 縦画面時の対応
- 上部に4:3のゲームエリアを配置（前作踏襲）
- 下部は前作の談話室を踏襲。余裕があれば改善を検討
- OBS等のブラウザソースでは上部のみを表示する想定（前作踏襲）

### パフォーマンス要件
- 画像プリロード戦略
- WebP形式の画像使用

### アクセシビリティ
- 対応ブラウザ: Chrome / Safari / Firefox / Edge
- コントラスト比4.5:1以上（前作踏襲）
- タッチターゲット44x44px相当以上（前作踏襲）
- 色覚多様性への配慮: 正解/不正解を色だけでなくテキストでも伝える（前作踏襲）
- クリック不可要素にホバーエフェクトをつけない（前作踏襲）
