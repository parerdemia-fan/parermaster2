# スモークテスト仕様

本ドキュメントはスモークテストの対象シナリオと検証項目を定義する。

## 前提条件

- 開発サーバーが `http://localhost:5173/parermaster2/` で起動済み（`pnpm dev`）
- Playwright MCP が利用可能

## テストシナリオ

### ST-01: タイトル画面の表示

- タイトル画面が正常に表示される
- メインボタン（1期生編、2期生編、寮別モード）が表示される
- サブメニュー（寮生一覧、相性診断、実績、案内）が表示される

### ST-02: 基本クイズフロー（2期生編）

1. タイトル画面で「2期生編」クリック → 設定画面に遷移する
2. 設定画面で「顔名前当て」を明示的に選択する（前回設定が残る場合があるため）
3. 設定画面でゲームモード選択肢（おぼえよう、顔名前当て、知識クイズ）が表示される
4. 「スタート！」クリック → クイズ画面に遷移する
5. クイズ画面で問題文・選択肢・進捗表示（X/Y形式）が表示される
6. 選択肢をクリックして回答できる
7. 全問回答後、結果画面に遷移する（最終問のボタンは「結果を見る」）
8. 結果画面でスコアが表示される

### ST-03: 画面遷移（サブメニュー）

以下の各画面について、タイトルからの遷移と復帰を確認する:

1. 寮生一覧 — タレントのリストが表示される
2. 相性診断 — 診断画面が表示される
3. 実績 — バッジ一覧が表示される
4. 案内 — ゲーム情報が表示される

### ST-04: おぼえようモード

1. タイトル画面で「2期生編」クリック → 設定画面
2. 「おぼえよう」を明示的に選択する
3. 「スタート！」クリック → おぼえよう画面に遷移する
4. 暗記フェーズが表示される（タレント画像と名前3人、「覚えた！」ボタン）

### ST-05: 相性診断フロー

1. タイトル画面で「相性診断」クリック → イントロ画面に遷移する
2. 「🌙 1期生との相性診断」または「🌸 2期生との相性診断」をクリック → 診断画面に遷移する
3. 10問の質問それぞれで選択肢をクリックして回答する
4. 全問回答後、診断結果画面に遷移する
5. 選択した世代のタレントのみがトップ3に出る
6. 「🔄 もう一度」でイントロ画面に戻り、別世代を試せる

### ST-06: タイムアタックフロー

前提: DEV画面のBadgeEditorでタイムアタック解放条件を満たす必要がある。

1. DEV画面でバッジを設定してタイムアタックを解放する
2. タイトル画面で「タイムアタック」ボタンが有効になっている
3. 「タイムアタック」クリック → 確認ダイアログが表示される
4. 「スタート！」クリック → クイズ画面に遷移する
5. カウントダウン後、タイマーが表示される
6. 数問回答して「やめる」→ 確認ダイアログ → タイトルに戻る
7. テスト後、DEV画面でバッジをリセットして元に戻す

## 問題タイプ別の回答方法

クイズの回答操作は問題タイプによって異なる:

| 問題タイプ | ヘッダー表示 | 回答操作 |
|-----------|------------|---------|
| 顔当て | 📸 顔当て | 4つの画像ボタン（`button "選択肢"`）の1つをクリック |
| 名前当て | ✏️ 名前当て | 4つのテキストボタン（`button "👤 名前"`）の1つをクリック |
| 名前を作ろう | 🧩 名前を作ろう | 名前パーツボタンを2つ選択 → 「OK」ボタンをクリック |
| テキストクイズ | 📝 知識クイズ | 4つのテキスト選択肢ボタンの1つをクリック |
| ぼかし | 🔍 ぼかし | 4つのテキストボタン（`button "👤 名前"`）の1つをクリック |
| スポットライト | 💡 スポットライト | 4つのテキストボタン（`button "👤 名前"`）の1つをクリック |

## タイムアタック解放方法（DEV画面）

DEV画面（タイトル画面の「DEV」ボタン）の BadgeEditor で以下のバッジを設定する:

| スロット | 必要ランク |
|---------|----------|
| gen2_all（2期生） | gold |
| gen2_knowledge（2期生・知識クイズ） | bronze |
| gen1_all（1期生） | gold |
| gen1_knowledge（1期生・知識クイズ） | gold |

## 視覚チェック観点

各画面のスクリーンショットに対して、以下の観点で目視チェックを行う:

### テキストの視認性
- 文字が背景に溶け込んでいないか（コントラスト不足）
- テキストが切れ・はみ出し・省略されていないか
- 文字サイズが小さすぎて読みにくくないか

### クリッカブル要素
- ボタンやタップ対象が小さすぎないか（指で押せるサイズか）
- クリック可能であることが視覚的に分かるか（形状・色・影等）
- 他の要素（立ち絵・装飾等）がボタンに重なって操作を阻害していないか

### レイアウトバランス
- 要素間の余白が詰まりすぎ・空きすぎていないか
- 要素の配置に偏りがないか（左右・上下のバランス）
- 画面内に不自然な空白エリアがないか

### 要素の重なり・切れ
- 画像やテキストが意図せず重なっていないか
- 画像が枠からはみ出したり、不自然にクリップされていないか
- スクロールしないと見えない重要な要素がないか

## 検証方法

- `browser_snapshot` でアクセシビリティツリーを取得し、要素の存在を確認する
- `browser_take_screenshot` で各画面のスクリーンショットを撮影する
- クリック対象は `browser_snapshot` の ref 値を使って `browser_click` で操作する
- 撮影したスクリーンショットを Read ツールで読み込み、視覚チェック観点に沿って分析する

## 注意事項

- ゲームモード設定は localStorage に永続化される。テスト時は必ず対象モードを明示的に選択すること
- 多数の問題を回答する場合（ST-02、ST-05）は `browser_evaluate` でループ処理を使って効率化する
- 回答後のナビボタンは「次へ →」、最終問のみ「結果を見る」
