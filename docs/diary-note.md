# 開発日誌 手記エントリ設計

## 概要

開発日誌にプレイヤーの行動が反映される動的な「手記エントリ」を追加する。AI（くろぴ）がプレイヤーの活動を観察して書いたような体裁で、プレイヤーごとに内容が異なる。

## 基本方針

- 1日の成果をまとめて**1エントリ**にする（バッジごとに1エントリではない）
- セッション中の出来事を LocalStorage に蓄積し、**次回アクセス時（翌日以降）**に手記エントリとして生成する
- 一度生成されたエントリは固定（再プレイで上書きしない）
- 開発記録（静的エントリ）と手記エントリを日付順にマージして表示する

## 蓄積する出来事

プレイ中に以下のイベントが発生したら LocalStorage に記録する。

| イベント | 記録する情報 |
|---------|------------|
| クイズ完了 | 日付、世代、モード、難易度、正解率 |
| パーフェクト達成 | 日付、世代、モード、難易度 |
| バッジ獲得/ランクアップ | 日付、バッジ名、ランク |
| 称号獲得 | 日付、称号名（パレ学マスター/グランドマスター等） |
| タイムアタック記録更新 | 日付、タイム |
| 初回プレイ | 日付、最初にプレイしたモード |

### LocalStorage 形式

```json
{
  "parermaster2_note_pending": {
    "date": "2026-04-04",
    "events": [
      { "type": "quiz_complete", "gen": "1期生", "mode": "顔名前当て", "diff": "ふつう", "rate": 72 },
      { "type": "perfect", "gen": "1期生", "mode": "顔名前当て", "diff": "ふつう" },
      { "type": "badge", "name": "顔名前当て", "rank": "ゴールド" },
      { "type": "ta_record", "time": 83450 }
    ]
  }
}
```

- `date` はその日最初のイベント発生時の日付（JST）
- 同じ日のイベントは `events` 配列に追記される
- 翌日以降のアクセス時に消費され、手記エントリに変換される

## 手記エントリの生成

### 生成タイミング

アプリ起動時に `parermaster2_note_pending` を確認し、`date` が今日より前であれば手記エントリを生成する。

### 生成ロジック

蓄積されたイベントからテンプレートベースでエントリ本文を組み立てる。

```
{date}の記録 ―

{player}さんが遊びに来てくれました。

{イベントに応じた本文}

明日もお待ちしています。
```

イベント種別ごとのテンプレート例：

| イベント | テンプレート例 |
|---------|--------------|
| quiz_complete | 「{gen} {mode} {diff}に挑戦して、{rate}%の正解率でした。」 |
| perfect | 「{gen} {mode} {diff}でパーフェクトを達成しました！」 |
| badge | 「{name}の{rank}バッジを手に入れました。」 |
| title | 「ついに{title}の称号を獲得しました……！」 |
| ta_record | 「タイムアタックで{time}の記録を出しました。」 |
| first_play | 「初めて遊びに来てくれました。最初に挑戦したのは{mode}でした。」 |

複数イベントがある場合はすべてを1つの本文にまとめる。順序はイベント発生順。

### 特別なエントリ

以下のイベントは通常より特別な文面にする：

- **初回プレイ**: 歓迎の気持ちを込めた特別な導入文
- **パレ学マスター獲得**: 感動的なまとめ
- **パレ学グランドマスター獲得**: 最大の到達点としての特別な文面

## 開発日誌との統合表示

### 同じ日に開発記録がある場合

開発記録エントリの本文の**末尾に区切り線を入れて手記を追記**する。タイトルは開発記録のタイトルをそのまま使用。

```
（開発記録の本文）

---

📝 {player}さんの手記

{手記の本文}
```

### 同じ日に開発記録がない場合

手記のみのエントリとして表示する。

- タイトル: 「{player}さんの冒険記録」
- 日付: イベント発生日
- 一覧での見た目: 開発記録エントリと区別する（アイコンや背景色等）

## データソース

- **開発記録（静的）**: `public/data/diary.json`（`pnpm sync` で生成）
- **手記エントリ（動的）**: LocalStorage に保存
  - 蓄積中: `parermaster2_note_pending`
  - 生成済み: `parermaster2_note_entries` （DiaryEntry[] 形式）

## 表示時のマージ

`useDiary` フックを拡張し、diary.json の静的エントリと LocalStorage の動的エントリを日付順にマージして返す。

```typescript
interface DiaryEntry {
  date: string
  title: string
  body: string
  type: 'dev' | 'note'  // 開発記録 or 手記
}
```

同じ日に両方ある場合は `type: 'dev'` のエントリの `body` に手記本文を追記し、1エントリとして扱う。

## 実装ステップ

1. イベント蓄積ロジック（LocalStorage への書き込み）
2. 手記エントリ生成ロジック（テンプレートベース）
3. `useDiary` の拡張（静的 + 動的のマージ）
4. `DiaryScreen` の表示調整（手記エントリの視覚的区別）
5. テンプレートの文面作成・調整
