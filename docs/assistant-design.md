# アシスタント設計

## 現状

クイズ画面のヘッダー右端に、1問ごとにランダムな1期生が表示される。
セリフは以下の9パターンのみで固定：

- **回答前**: 問題タイプ別に1パターンずつ（計7パターン）
- **正解時**: 「すごい！正解だよ〜！」（1パターン）
- **不正解時**: 「あちゃ〜、残念！」（1パターン）

口調は全タレント共通。

## 改善方針

**グループ口調方式（方式C）を採用。**

- タレントを口調グループに分類
- グループごとにセリフセットを用意
- 基本グループ＋キャラ固有グループの構成で、随時追加可能

## 口調グループ

### 基本グループ（初期6グループ）

| グループID | 名前 | 語尾・特徴 | 正解時の例 | 不正解時の例 |
|-----------|------|-----------|-----------|------------|
| `polite` | 丁寧語 | 〜です、〜ますね | 「正解です！すごいですね！」 | 「残念です…次がんばりましょう！」 |
| `gal` | ギャル | 〜じゃん、まじ、やば | 「まじ正解じゃん！やば〜！」 | 「え〜うそ〜、残念じゃん！」 |
| `wild` | ワイルド | 〜だぜ、〜だろ | 「正解だぜ！やるじゃん！」 | 「ドンマイだぜ！次いこうぜ！」 |
| `ojou` | お嬢様 | 〜ですわ、〜ですこと | 「正解ですわ！お見事ですこと！」 | 「残念ですわ…次は頑張って下さいまし」 |
| `gentle` | おっとり | 〜だよぉ、ふわっと | 「わぁ…正解だぁ…すごいねぇ」 | 「あぅ…残念だねぇ…」 |
| `genki` | 元気 | ！多め、テンション高い | 「正解！！やったね！！」 | 「ドンマイドンマイ！！次！！」 |
| `kansai` | 関西弁 | 〜やん、〜やで | 「正解やん！すごいやん！」 | 「あちゃ〜、残念やなぁ」 |

### キャラ固有グループ（随時追加）

特定のタレント専用の口調。基本グループに収まらない個性的なキャラ向け。

- グループIDは自由（例: `cool_reina`, `chuuni_xxx` 等）
- 基本グループと同じセリフ構造を持つ
- タレント1人だけに割り当ててもよいし、複数人で共有してもよい

口調が不明なタレントは `polite`（丁寧語）をデフォルトとする。

## セリフデータ構造

### セリフの種類

| カテゴリ | キー | タイミング | 備考 |
|---------|------|-----------|------|
| 回答前 | `before:{typeId}` | 問題表示時 | 問題タイプ別（7タイプ） |
| 正解 | `correct` | 正解回答後 | |
| 不正解 | `wrong` | 不正解回答後 | |

各カテゴリ × 口調グループごとに **3〜5パターン** のセリフ配列を用意。

### 必要セリフ数の見積もり

- 1グループあたり: (7タイプ × 3パターン) + (正解5) + (不正解5) = **31セリフ**
- 基本7グループ: 31 × 7 = **約220セリフ**
- キャラ固有グループは追加分

## データ管理

### スプレッドシート

2つのシートで管理：

**シート1: タレント口調割り当て**

| タレントID | 口調グループID |
|-----------|--------------|
| 25CO002 | genki |
| 25ME001 | ojou |
| 25WA004 | polite |
| ... | ... |

→ `pnpm sync` で `talents.json` の各タレントに `speechGroup` フィールドとして反映

**シート2: セリフデータ**

| 口調グループID | カテゴリ | セリフ1 | セリフ2 | セリフ3 | セリフ4 | セリフ5 |
|--------------|---------|--------|--------|--------|--------|--------|
| polite | before:face-guess | この子の顔、わかりますか？ | 見覚えありますか？ | どの子でしょう？ | | |
| polite | correct | 正解です！すごいですね！ | さすがです！ | お見事です！ | 大正解ですね！ | やりましたね！ |
| polite | wrong | 残念です… | 惜しいです！ | 次がんばりましょう！ | ドンマイです！ | |
| gal | before:face-guess | この子の顔わかる〜？ | 見たことある〜？ | どの子かわかる？ | | |
| ... | ... | ... | ... | ... | ... | ... |

→ `pnpm sync` で `public/data/assistant-comments.json` として出力

### JSON出力形式

```json
{
  "polite": {
    "before:face-guess": ["この子の顔、わかりますか？", "見覚えありますか？", "どの子でしょう？"],
    "before:name-guess": ["この子の名前は？", "誰だかわかりますか？", "名前出てきますか？"],
    "correct": ["正解です！すごいですね！", "さすがです！", "{player}さん、お見事です！"],
    "wrong": ["残念です…", "惜しいです！", "次がんばりましょう！", "{player}さん、ドンマイです！"]
  },
  "gal": {
    "before:face-guess": ["この子の顔わかる〜？", "見たことある〜？"],
    "correct": ["まじ正解じゃん！", "やば〜すご！"],
    "wrong": ["え〜残念〜", "うそ〜まじか〜"]
  }
}
```

## プレイヤー名の差し込み

セリフに `{player}` プレースホルダを含めることで、プレイヤー名入りのセリフを実現する。

### ルール

- `{player}` を含むセリフは各カテゴリのバリエーションの**一部**として混ぜる（全セリフに入れない）
- 例: `["正解です！", "さすがです！", "{player}さん、お見事です！"]` → 3回に1回程度で名前入り
- 表示時に `{player}` をプレイヤー名に置換する
- **はみ出し防止**: 置換後の文字数が上限（例: 20文字）を超える場合、そのセリフを破棄して `{player}` を含まない別のセリフを再選択する
- プレイヤー名が未設定の場合は `{player}` を含むセリフを選択候補から除外する

## 実装方針

1. `talents.json` に `speechGroup` フィールドを追加（sync対応）
2. `assistant-comments.json` を `pnpm sync` で生成
3. QuizHeader.tsx のセリフ選択ロジックを変更:
   - アシスタント選出時にタレントの `speechGroup` を取得
   - `assistant-comments.json` から `speechGroup × カテゴリ → セリフ配列` を引く
   - 配列からランダムに1つ選択
   - `{player}` を含む場合はプレイヤー名で置換し、文字数上限チェック
4. フォールバック: `speechGroup` 未設定 → `polite`、セリフ配列が空 → 現在のハードコードセリフ

## 状況連動セリフ（将来拡張）

優先度は低いが、拡張の余地として記録：

- **連続正解時**: 「3連続正解！絶好調！」
- **連続不正解時**: 「大丈夫、落ち着いて！」
- **残り数問時**: 「あと少し！」「ラストスパート！」
- **タイムアタック残り時間少**: 「急いで！」

カテゴリキーを追加（`streak:3`, `last_few` 等）すれば同じ仕組みで対応可能。
