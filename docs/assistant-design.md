# アシスタント設計

## 現状

クイズ画面のヘッダー右端に、1問ごとにランダムな1期生が表示される。
セリフは以下の9パターンのみで固定：

- **回答前**: 問題タイプ別に1パターンずつ（計7パターン）
- **正解時**: 「すごい！正解だよ〜！」（1パターン）
- **不正解時**: 「あちゃ〜、残念！」（1パターン）

口調は全タレント共通。

## 改善方針

**グループ口調方式（方式C）を採用。**

- タレントを口調グループに分類
- グループごとにセリフセットを用意
- 基本グループ＋キャラ固有グループの構成で、随時追加可能

## 口調グループ

### 基本グループ（初期6グループ）

| グループID | 名前 | 語尾・特徴 | 正解時の例 | 不正解時の例 |
|-----------|------|-----------|-----------|------------|
| `polite` | 丁寧語 | 〜です、〜ますね | 「正解です！すごいですね！」 | 「残念です…次がんばりましょう！」 |
| `gal` | ギャル | 〜じゃん、まじ、やば | 「まじ正解じゃん！やば〜！」 | 「え〜うそ〜、残念じゃん！」 |
| `wild` | ワイルド | 〜だぜ、〜だろ | 「正解だぜ！やるじゃん！」 | 「ドンマイだぜ！次いこうぜ！」 |
| `ojou` | お嬢様 | 〜ですわ、〜ですこと | 「正解ですわ！お見事ですこと！」 | 「残念ですわ…次は頑張って下さいまし」 |
| `gentle` | おっとり | 〜だよぉ、ふわっと | 「わぁ…正解だぁ…すごいねぇ」 | 「あぅ…残念だねぇ…」 |
| `genki` | 元気 | ！多め、テンション高い | 「正解！！やったね！！」 | 「ドンマイドンマイ！！次！！」 |
| `kansai` | 関西弁 | 〜やん、〜やで | 「正解やん！すごいやん！」 | 「あちゃ〜、残念やなぁ」 |

### キャラ固有グループ（随時追加）

特定のタレント専用の口調。基本グループに収まらない個性的なキャラ向け。

- グループIDは自由（例: `cool_reina`, `chuuni_xxx` 等）
- 基本グループと同じセリフ構造を持つ
- タレント1人だけに割り当ててもよいし、複数人で共有してもよい

口調が不明なタレントは `polite`（丁寧語）をデフォルトとする。

## セリフデータ構造

### セリフの種類

| カテゴリ | キー | タイミング | 備考 |
|---------|------|-----------|------|
| 回答前 | `before:{typeId}` | 問題表示時 | 問題タイプ別（7タイプ） |
| 正解 | `correct` | 正解回答後 | |
| 不正解 | `wrong` | 不正解回答後 | |

各カテゴリ × 口調グループごとに **3〜5パターン** のセリフ配列を用意。

### 必要セリフ数の見積もり

- 1グループあたり: (7タイプ × 3パターン) + (正解5) + (不正解5) = **31セリフ**
- 基本7グループ: 31 × 7 = **約220セリフ**
- キャラ固有グループは追加分

## データ管理

### スプレッドシート

2つのシートで管理（`sync-config.json` 参照）：

**プロフィールシート（1期生/2期生）の `tone` 列**

各タレントの口調グループ名が入る。空欄の場合は「丁寧語」として扱う。

| (他の列) | tone |
|---------|------|
| ... | 元気 |
| ... | ギャル |
| ... | （空欄 → 丁寧語） |

→ `pnpm sync` で `talents.json` の各タレントに `tone` フィールドとして反映

**セリフシート（`quotes`）**

口調タイプ列には**口調グループ名**または**タレント名**が入る。

| 口調タイプ | 場面 | セリフ1 | セリフ2 | セリフ3 | セリフ4 | セリフ5 |
|-----------|------|--------|--------|--------|--------|--------|
| 丁寧語 | 顔当て | この子の顔、わかりますか？ | 見覚えありますか？ | どの子でしょう？ | | |
| 丁寧語 | 正解時 | 正解です！すごいですね！ | さすがです！ | お見事です！ | | |
| ギャル | 正解時 | まじ正解じゃん！🎉 | やば〜すご！✨ | 天才じゃん！ | | |
| 山田花子 | 正解時 | やったー！花子うれしい！ | 花子も正解だと思ってた！ | | | |

→ `pnpm sync` で `public/data/quotes.json` として出力

#### 口調タイプ列がタレント名の場合

セリフシートの口調タイプ列にタレント名が入っている行は、そのタレント固有のセリフとして扱う。実行時にはそのタレントの口調グループのセリフと**マージ**される。

**例: 山田花子（口調: ギャル）が正解時のアシスタントになった場合**

1. ギャルの正解時セリフ: `["まじ正解じゃん！🎉", "やば〜すご！✨", "天才じゃん！"]`
2. 山田花子の正解時セリフ: `["やったー！花子うれしい！", "花子も正解だと思ってた！"]`
3. 候補 = 1 + 2 の合算: **5パターン**からランダム選択

これにより、タレント固有のセリフがあっても口調グループの汎用セリフも引き続き使われ、固有セリフが少なくてもバリエーションが確保される。

### JSON出力形式

```json
{
  "groups": {
    "丁寧語": {
      "顔当て": ["この子の顔、わかりますか？", "見覚えありますか？", "どの子でしょう？"],
      "正解時": ["正解です！すごいですね✨", "さすがです！👏", "{player}さん正解！✨"],
      "不正解時": ["残念です…😢", "惜しいですね！", "{player}さん💦"]
    },
    "ギャル": {
      "顔当て": ["この子の顔わかる〜？", "見たことある〜？"],
      "正解時": ["まじ正解じゃん！🎉", "やば〜すご！✨"],
      "不正解時": ["え〜残念〜😭", "うそ〜まじか〜"]
    }
  },
  "talents": {
    "山田花子": {
      "正解時": ["やったー！花子うれしい！", "花子も正解だと思ってた！"]
    }
  }
}
```

## プレイヤー名の差し込み

セリフに `{player}` プレースホルダを含めることで、プレイヤー名入りのセリフを実現する。

### ルール

- `{player}` を含むセリフは各カテゴリのバリエーションの**一部**として混ぜる（全セリフに入れない）
- 例: `["正解です！", "さすがです！", "{player}さん、お見事です！"]` → 3回に1回程度で名前入り
- 表示時に `{player}` をプレイヤー名に置換する
- **はみ出し防止**: 置換後の文字数が上限（例: 20文字）を超える場合、そのセリフを破棄して `{player}` を含まない別のセリフを再選択する
- プレイヤー名が未設定の場合は `{player}` を含むセリフを選択候補から除外する

## 実装方針

1. `talents.json` に `tone` フィールドを追加（sync対応、空欄時は「丁寧語」）
2. `quotes.json` を `pnpm sync` で生成（`groups` + `talents` の2階層）
3. QuizHeader.tsx のセリフ選択ロジックを変更:
   - アシスタント選出時にタレントの `tone` と `name` を取得
   - `groups[tone][場面]` のセリフ配列を取得
   - `talents[name][場面]` が存在すればそのセリフ配列をマージ
   - マージ後の配列からランダムに1つ選択
   - `{player}` を含む場合はプレイヤー名で置換し、文字数上限チェック
4. フォールバック: `tone` 未設定 → 「丁寧語」、セリフ配列が空 → 現在のハードコードセリフ

## 状況連動セリフ（将来拡張）

優先度は低いが、拡張の余地として記録：

- **連続正解時**: 「3連続正解！絶好調！」
- **連続不正解時**: 「大丈夫、落ち着いて！」
- **残り数問時**: 「あと少し！」「ラストスパート！」
- **タイムアタック残り時間少**: 「急いで！」

カテゴリキーを追加（`streak:3`, `last_few` 等）すれば同じ仕組みで対応可能。
